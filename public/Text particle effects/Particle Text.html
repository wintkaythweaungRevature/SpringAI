Type that knows what it means.

Ember & Aether is a lightweight runtime that recognises semantic words inside any text field and renders matching particle, light and material effects — live, per-character, directly behind the glyphs. No shaders to write. No assets to ship.

// install
npm i ember-aether

// usage
import { bind } from 'ember-aether'
bind(document.querySelector('textarea'))

— 14kb gzipped · zero deps
01 / ENGINE

Per-word physics

Each recognised token spawns an independent simulation with its own forces, palette and lifetime.

02 / TYPE

Glyph-aware

Effects anchor to actual rendered glyph boxes — they follow resizing, wrapping and edits in real time.

03 / API

One-line binding

Attach to any textarea, contenteditable or input. Customise the dictionary, or bring your own words.

04 / PERF

60fps by default

A shared canvas pool caps particle counts and pauses simulations that drift off-screen.
